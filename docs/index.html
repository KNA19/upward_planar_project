About the Dataset

This page provides datasets generated by exhaustive computation over small point-set order types. For each value of n, the input point sets are representatives of order types from Oswin Aichholzer’s order-type database. Each point set is treated as a fixed set of indexed points, and the dataset records which path embeddings and orientations admit an upward planar straight-line drawing.

For each representative point set, we enumerate undirected straight-line path embeddings and keep one representative up to path reversal. This means that the order [v0, v1, ..., v_{n-1}] and its reverse [v_{n-1}, ..., v1, v0] are treated as the same undirected embedded path. Each such path is first tested for planarity, and only plane path embeddings are stored.

For every stored undirected plane path, the dataset reports the directed orientations that become upward after a suitable rotation of the coordinate system. Thus, the point coordinates are stored in their original representative order-type coordinate system, while the fields up_direction_angle and rotation_angle provide witness information for viewing the selected orientation as upward.

The purpose of this archive is to make the computational data easy to download, inspect, and verify. The full JSON files contain the complete embedding data, while the summary files provide compact counts and metadata. A lightweight browser is also included for inspecting individual examples without requiring users to run the full visualization app.

Data Format

Each full dataset file is organized by point set. A point-set record contains the coordinates of the representative point set and a list of undirected plane path embeddings. Each plane path then contains the upward orientations found for that path.

The field order gives the undirected path as a sequence of point indices. The field dirs assigns directions to the consecutive edges of this order. A plus sign indicates that the edge follows the listed order, while a minus sign indicates that the edge is directed in the reverse direction.

Upwardness is considered up to rotation of the coordinate system. Therefore, a drawing may not appear upward in the original coordinate view. The rotation_angle field gives a witness rotation for displaying the selected orientation as an upward drawing.

points

Coordinates of the representative point set. Point indices are determined by the order of this list.

plane_paths

Undirected plane path embeddings for the selected point set, stored once up to path reversal.

order

The sequence of point indices along the undirected path. For example, [0, 2, 1, 3] means that the path visits point 0, then point 2, then point 1, then point 3.

upward_orientations

The list of orientations of the selected plane path that become upward after rotating the coordinate system.

dirs

Direction string for the path edges. + means order[i] → order[i+1], and - means order[i+1] → order[i].

rotation_angle

A witness rotation angle in radians. Rotating the drawing by this angle displays the selected orientation as upward.

up_direction_angle

A witness upward direction in the original coordinate system, measured in radians.

summary files

Compact JSON files containing the number of point sets, plane paths, and upward orientations for each value of n.

full files

Complete JSON files containing all point sets, plane paths, and upward orientations for the corresponding value of n.

Full schema file: data/schema.json
