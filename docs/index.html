About the Dataset

This page provides the full witness datasets generated from representative point-set order types. The input point sets come from Oswin Aichholzer’s order-type database (source page). Each point set is treated as a fixed set of indexed points, and the dataset records an explicit upward planar straight-line witness for every directed path-orientation pattern covered by the computation.

The dataset family is stored in outputs_witness and covers n = 3, 4, 5, 6, 7, 8, 9, 10. For each value of n, the required orientation patterns are all binary strings of length n - 1. The files therefore range from 4 required patterns for n = 3 to 512 required patterns for n = 10.

Each witness stores the point coordinates, the selected path order, the undirected edges, the directed edges, and the rotation data needed to display the drawing as upward. The point coordinates remain in their original representative order-type coordinate system, while up_direction_angle and rotation_angle provide the witness information for viewing the same drawing in an upward direction.

The purpose of this archive is to make the computational data easy to download, inspect, and verify. The JSON files contain the full witness data, and the lightweight browser below is limited to the new witness dataset.

Data Format

Each full witness dataset file contains metadata for one value of n and a top-level witnesses list. Each witness is one explicit upward planar straight-line embedding for a directed orientation pattern.

The field order gives the path as a sequence of point indices. The field dirs assigns directions to the consecutive edges of this order. A plus sign indicates that the edge follows the listed order, while a minus sign indicates that the edge is directed in the reverse direction.

Upwardness is considered up to rotation of the coordinate system. Therefore, a drawing may not appear upward in the original coordinate view. The rotation_angle field gives a witness rotation for displaying the selected orientation as an upward drawing.

witnesses

The list of explicit witnesses, one for each covered orientation pattern.

points

Coordinates of the representative point set used by this witness. Point indices are determined by the order of this list.

order

The sequence of point indices along the undirected path. For example, [0, 2, 1, 3] means that the path visits point 0, then point 2, then point 1, then point 3.

directed_edges

Directed edge list for the witness drawing, stored as [tail, head] pairs.

dirs

Direction string for the path edges. + means order[i] → order[i+1], and - means order[i+1] → order[i].

rotation_angle

A witness rotation angle in radians. Rotating the drawing by this angle displays the selected orientation as upward.

up_direction_angle

A witness upward direction in the original coordinate system, measured in radians.

undirected_edges

Undirected path edges for the selected path order.

complete

Indicates whether witnesses were found for all required orientation patterns for the corresponding value of n.
